# SinoraX
[![MIT License](https://img.shields.io/badge/license-MIT-blue.svg)](./LICENSE)

> AI is not something you use — it’s something you assemble. Build your fully user-owned hybrid AI stack.

## Short Description
Local-first composable AI gateway unifying local LLMs & cloud APIs with BYO/BYOK/BYOM principles; a privacy-centric personal intelligence orchestration layer.

## 🚀 Core Vision
Today’s AI ecosystem exists as fragmented closed silos: GPT, Claude, Gemini, Ollama and other local runtimes operate in isolation. Switching between models requires juggling separate API keys, disjointed workflows and locked platform ecosystems.

SinoraX rethinks this paradigm entirely.
Instead of treating AI as a single fixed product users must choose, this project aims to build a customizable hybrid intelligence layer that bridges offline local inference and cloud large model services within one unified interface.

The long-term shift we’re building toward: from closed AI platforms to user-assembled personal AI stacks.

## 🧩 Three Foundational Design Principles (Unique to SinoraX)
The entire architecture is guided by three user-sovereignty rules that separate this project from generic multi-model clients:
1. **BYO (Bring Your Own)**
You fully own and define your AI runtime environment, with zero platform vendor lock-in.
2. **BYOK (Bring Your Own Key)**
Supply your own API credentials for every cloud provider; no mandatory billing ties or third-party payment intermediaries.
3. **BYOM (Bring Your Own Model)**
Seamlessly integrate self-hosted, local open-source LLMs alongside commercial cloud models under one system.

Combined, these principles enable a fully hybrid AI experience balancing offline privacy and high-performance cloud compute.

## Key Planned Capabilities
- Single unified interface compatible with local inference engines (Ollama, LM Studio, OpenAI-compatible local runtimes)
- Native support for mainstream cloud LLM APIs (GPT, Gemini, Groq, OpenRouter, etc.)
- Modular model orchestration: every LLM provider functions as a swappable component
- Intelligent request routing & automatic fallback logic for slow/unavailable models
- Fully local-first privacy design: all user chat data, prompts and preferences stay client-side
- Lightweight frontend architecture with no mandatory central backend server dependency
- Built-in developer visibility for routing, latency and model usage metrics
- Full offline operation when local models are available

## 🎯 What Makes SinoraX Different
Most multi-model AI tools only aggregate chat endpoints.
SinoraX is designed as an orchestration layer — analogous to microservice infrastructure, but built for artificial intelligence. Models are no longer rigid fixed endpoints; they become interchangeable compute units users mix and match based on privacy requirements, cost, speed and personal preference.
